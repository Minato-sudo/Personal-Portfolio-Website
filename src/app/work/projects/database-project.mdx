## Overview

This project focused on designing and implementing a structured database-backed web system rather than treating the database as an afterthought. Talha organized the application around relational entities, user-facing templates, and dashboard flows that made the data model visible in the experience itself.

## What Was Built

- A relational schema in SQL to manage core entities and their relationships in a consistent, queryable structure.
- Multiple EJS interface views including login, account, dashboard, and judge-facing pages connected to the underlying database logic.
- An ERD-led design workflow that translated requirements into tables, keys, and navigable application flows.

## Why It Matters

The project demonstrates a strong systems mindset: designing the data model, interface structure, and query paths together. That makes it a useful example of Talha's ability to work across software engineering, relational thinking, and practical application architecture.

## Stack

- SQL
- MySQL
- Node.js
- EJS
- ERD
- Relational Mapping

## Outcome

The result is a compact but well-structured database application that showcases schema planning, multi-page integration, and the discipline required to turn raw entities into a usable software system.
